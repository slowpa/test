 GitHub 仓库中文件和文件夹命名（例如 `.md`、`.js`、`src`、`site`）都是软件开发和项目管理中**约定俗成的规范**。

GitHub 本身对文件和文件夹的命名**没有强制规定**（即您爱怎么命名都行），但为了项目清晰、方便他人协作和工具识别，社区形成了一些**强烈推荐的命名习惯**。

---

## 📘 常见文件扩展名 (文件类型)

文件扩展名是告诉电脑和人“这个文件是干什么的”的关键：

| 扩展名                     | 含义             | 用途                                         |
|:----------------------- |:-------------- |:------------------------------------------ |
| **`.md`** / `.markdown` | **Markdown**   | 用于文档、博客文章、说明文档（如 `README.md`）。这是您写书评的最佳选择。 |
| **`.js`** / `.jsx`      | **JavaScript** | 用于网页前端（浏览器端）或 Node.js 后端的编程逻辑。             |
| **`.css`** / `.scss`    | **CSS / Sass** | 用于定义网页的样式、颜色、布局和外观。                        |
| **`.html`**             | **HTML**       | 用于定义网页的结构和内容（如标题、段落、图片）。                   |
| **`.py`**               | **Python**     | 用于后端开发、数据分析、脚本自动化等。                        |
| **`.json`** / `.xml`    | **数据格式**       | 用于存储结构化数据，常用于配置和数据交换。                      |
| **`.yaml`** / `.yml`    | **配置格式**       | 常用于软件配置和自动化流程（如 GitHub Actions）。           |

## 📁 常见文件夹命名规范 (项目结构)

文件夹的命名旨在提供清晰的**逻辑分离**，告诉协作者和工具代码或资源的作用。

| 文件夹名                  | 含义                        | 约定俗成的用途                                             |
|:--------------------- |:------------------------- |:--------------------------------------------------- |
| **`src`** / `source`  | **Source Code** (源代码)     | 存放**项目的主要、核心**的源代码文件（如 `.js`, `.py` 等）。             |
| **`dist`** / `build`  | **Distribution** (发布/构建)  | 存放**经过编译、压缩和优化**后，可以直接部署和发布给用户使用的最终文件。              |
| **`docs`**            | **Documentation** (文档)    | 存放项目的详细说明、指南、教程和 API 文档等，**不属于代码本身**。               |
| **`site`** / `public` | **Website/Public Assets** | 存放网站静态文件（如图片、图标、CSS 文件），通常是 GitHub Pages 等服务部署的源目录。 |
| **`test`** / `tests`  | **Tests** (测试)            | 存放用于验证代码功能的自动化测试文件。                                 |
| **`assets`**          | **Assets** (资源)           | 存放图片、字体、视频等媒体资源文件。                                  |

## ⭐️ 必须要遵守的特殊文件

虽然大部分命名是约定俗成的，但有些文件对 GitHub 或其他工具而言具有**特殊意义**，命名是固定的：

| 文件名                | 作用                                                  | 必须遵守吗？            |
|:------------------ |:--------------------------------------------------- |:----------------- |
| **`README.md`**    | **仓库主页**。GitHub 会自动在仓库首页展示它的内容，作为项目的介绍。             | **必须**以此命名。       |
| **`LICENSE`**      | **许可证文件**。定义了其他人使用您代码的权限和限制。                        | **必须**以此命名。       |
| **`package.json`** | **Node.js/前端配置**。列出项目依赖的库和脚本。                       | 特定项目**必须**以此命名。   |
| **`.gitignore`**   | **Git 忽略文件**。告诉 Git **不要**跟踪和上传哪些文件（如临时文件、密码、编译产物）。 | **必须**以此命名（以点开头）。 |

### 总结

对于您要做的书评仓库：

1. 您的书评文件应该使用 **`.md`** 扩展名。
2. 您可以创建一个名为 **`reviews`** 或 **`posts`** 的文件夹来存放所有的 `.md` 文件。
3. 您**必须**拥有一个 **`README.md`** 文件作为目录或介绍。

这些命名规范（如 `src`、`.js`）的目的是让一个新开发者一看就知道这个项目是干什么的，内容放在哪里，极大地提高了协作效率。
